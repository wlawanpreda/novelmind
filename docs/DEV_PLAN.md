## 0. สถานะปัจจุบัน (ประเมินตรงไปตรงมา)

**แข็งแรงแล้ว** — pipeline ครบวงจร idea→scout→analyze→write→cover→audio→teaser→publish, รันต่อเนื่องได้ (orchestrator + launchd), คุมต้นทุน (cap + single-flight), routing hybrid (Gemini↔Mac mini), gateway HTTP service, web dashboard

**ตัวเลขจริงจาก production:**
| ขั้น | จำนวน | อ่านได้ว่า |
|------|------|-----------|
| เขียนแล้ว (Processed) | 36 เรื่อง · 49 ตอน | เขียนเยอะ แต่ส่วนใหญ่ตอนเดียว |
| ปก (cover) | 14 | ~60% ของเรื่องไม่มีปก |
| หนังสือเสียง (audio) | 8 | ~78% ไม่มีเสียง |
| teaser | 8 | ~78% ไม่มีคลิป |
| **เผยแพร่ (published)** | **0** | **ไม่เคยปล่อยของจริงเลย** |

### 🎯 ข้อค้นพบหลัก (ตัวกำหนดแผนทั้งหมด)
**ระบบ "เขียนเยอะ แต่ปิดจบ/ปล่อยของน้อย" — funnel พังท่อนปลาย** ไม่ใช่ปัญหาต้นน้ำ
(scout/analyze/write ดีเกินพอแล้ว) แต่:
1. เรื่องที่เขียนไม่ถูกแปลงเป็นสินทรัพย์พร้อมปล่อย (ปก/เสียง/teaser ตามไม่ทัน)
2. ไม่เคยเผยแพร่จริง → Phase 5 feedback ไม่มีข้อมูลจริงให้เรียนรู้ → วงจรโตไม่หมุน
3. เรื่องส่วนใหญ่ตอนเดียว → ไม่มีนิยายจบจริงให้คนติดตาม

## 1. เป้าหมายเชิงกลยุทธ์

ยกระดับจาก **"โรงงานผลิตต้นฉบับ"** → **"โรงงานที่ปล่อยของจริง เรียนรู้ และโตเอง"**

3 เสาหลัก:
- **(A) ปิด funnel ท่อนปลาย** — ทุกเรื่องที่คุ้มค่าต้องไปถึง "คลิปพร้อมปล่อย" และ "ปล่อยจริง"
- **(B) ปิดวงจรเรียนรู้ด้วยข้อมูลจริง** — ปล่อย→วัดผล→เรียนรู้→ป้อนกลับ (Phase 5 ที่มีเลือดจริง)
- **(C) รัน 24/7 ไม่ต้องเฝ้า** — เสถียร กู้ตัวเอง แจ้งเตือนเมื่อพัง

## 2. แผนพัฒนาเป็นเฟส

แท็ก: `[code]` ผมทำได้เลย · `[config]` ตั้งค่า/เครื่อง · `[decision]` รอคุณตัดสินใจ · `[user]` ต้องใช้ credential/บัญชีคุณ

### 🔴 NOW — สัปดาห์นี้ถึง 2 สัปดาห์ (ปลดบล็อก + ปิด funnel)

**A1. ปลดบล็อก image gateway** `[config]`
ตั้ง `IMAGE_BACKEND=local` (หรือ hybrid) บนเครื่อง gateway แล้ว restart → ปกเรนเดอร์ผ่าน ComfyUI ฟรี (พิสูจน์แล้วว่าทำงาน) แทน Imagen ที่โควต้าหมด

**A2. ทำเรื่องที่เขียนแล้วให้ครบสินทรัพย์** `[code]`
- คำสั่ง `./ansre finish` ใหม่: หาเรื่อง Processed ที่ "ขาดปก/เสียง/teaser" แล้วเติมให้ครบทีละเรื่อง (idempotent, ข้ามที่มีแล้ว)
- เป้าหมาย: 36 เรื่อง → มีปก+เสียง+teaser ครบทุกเรื่องที่ market_fit ผ่าน gate
- แก้ orchestrator ให้ cover/audio/teaser ตามทันงานเขียน (เดินครบทุกเรื่อง ไม่ใช่แค่เรื่องล่าสุด)

**A3. เสถียรภาพ Mac mini / เครือข่าย** `[code+config]`
- ปัญหา: Tailscale hostname resolve ไม่ได้บ่อย → ทุกอย่างตกมา Gemini
- แก้: ใส่ LAN IP เป็น fallback อัตโนมัติใน llm_provider/image_provider (ลอง Tailscale → ถ้าล่ม ลอง LAN IP → ถ้าล่ม Gemini)
- `./ansre doctor` เตือนชัดเมื่อ local ใช้ไม่ได้ + เปิด `ANSRE_CALL_GAP=6` เป็น default เมื่อ Mac mini ดับ

**A4. ปล่อยของจริงครั้งแรก (unblock Phase 5)** `[user+decision]`
- เลือกแพลตฟอร์มแรก (แนะนำ TikTok หรือ YouTube Shorts — แนวตั้ง 9:16 ตรงกับ teaser)
- ใส่ credential (ดู PUBLISHING.md) → `PUBLISH_TIKTOK=1` หรือ `PUBLISH_YOUTUBE=1`
- ปล่อย teaser ที่ดีที่สุด 3-5 คลิป → เริ่มมีข้อมูลจริง

### 🟡 NEXT — 1 ถึง 2 เดือน (คุณภาพ + วงจรเรียนรู้จริง)

**B1. เรือธงหลายตอน + คุณภาพต่อเนื่อง** `[code]`
- `./ansre flagship <title>` — ปั้นเรื่อง market_fit สูงให้ครบ 5-10 ตอนอัตโนมัติ
- **Story Bible/continuity**: ติดตามตัวละคร/พล็อต/กฎโลก ข้ามตอน กันขัดแย้ง (ตอนนี้แต่ละตอนรู้แค่ outline + ตอนก่อน)
- เลือก 2-3 เรื่องเด่นทำเป็นนิยายจบจริง → มีของให้คนติดตาม

**B2. Auto-QA เนื้อหา (LLM judge)** `[code]`
- หลังเขียนแต่ละตอน ให้ AI ตรวจ: ความต่อเนื่อง, โทน, ความยาว, จุดขัดแย้งกฎ, meta-talk leak
- คะแนนต่ำ → เขียนใหม่อัตโนมัติ (ไม่เกิน N รอบ) · เก็บคะแนนคุณภาพใน frontmatter

**B3. Phase 5 มีเลือดจริง** `[code+user]`
- ดึง analytics จริงจากแพลตฟอร์ม (YouTube/TikTok API) → `feedback record` อัตโนมัติ (ไม่ต้องกรอกมือ)
- A/B test: ชื่อเรื่อง/ปก/hook หลายแบบต่อเรื่อง → วัดว่าแบบไหนปัง → feed กลับ
- รายงาน "สูตรที่ปัง" อัปเดตอัตโนมัติ → bias scout/analyze/ideation

**B4. Scout ฉลาดขึ้น** `[code]`
- ตรวจซ้ำข้ามรอบ (cross-run dedup ด้วย fingerprint เนื้อหา ไม่ใช่แค่ id)
- ตรวจความเสี่ยงลิขสิทธิ์/ความคล้ายต้นฉบับก่อนเขียน (originality guard)
- เพิ่มช่องทาง + จับ "เรื่องกำลังมาแรง" (เรตการเติบโต ไม่ใช่ยอดสะสม)

**B5. Observability + แจ้งเตือน** `[code]`
- เก็บ metric รายขั้น (สำเร็จ/ล้ม/เวลา/ค่าใช้จ่าย) → หน้า dashboard "สุขภาพ pipeline"
- แจ้งเตือนเมื่อ: worker ล่ม, cap ใกล้เต็ม, error ติดต่อกัน, Mac mini ดับ (ผ่าน LINE/Discord webhook)

### 🟢 LATER — 3 เดือนขึ้นไป (สเกล + ธุรกิจ)

**C1. ผลิตขนาน + ตารางปล่อย** `[code]`
- คิวปล่อยอัตโนมัติ (ปล่อยวันละ N คลิป เวลาที่ engagement ดี)
- ผลิตหลายเรื่องขนานโดยไม่ชนทรัพยากร (ต่อยอด single-flight + worker pool)

**C2. หลายแพลตฟอร์ม + หลายรูปแบบ** `[code+user]`
- YouTube + TikTok + (นิยายแพลตฟอร์มไทย: ธัญวลัย/readAwrite/Fictionlog)
- แปลง 1 เรื่อง → หลายฟอร์แมต (teaser สั้น, ตอนเสียงเต็ม, โพสต์ตัวอักษร)

**C3. การเงิน/ROI** `[code]`
- ติดตามรายได้ต่อเรื่อง (ad/view/subscriber) เทียบต้นทุน LLM → ROI ต่อเรื่อง
- จัดสรรงบผลิตไปที่แนวที่ ROI ดี (ต่อยอด feedback loop เป็นการตัดสินใจเชิงธุรกิจ)

**C4. ทำให้ส่งต่อ/ขายได้** `[code]`
- one-command setup, เอกสารครบ, backup SecondBrain อัตโนมัติ, มี CI

## 3. ลำดับแนะนำ (ตาม dependency + ผลกระทบ)

```
สัปดาห์ 1:  A1 (image config) → A2 (finish assets) → A3 (network เสถียร)
            └─ ผล: 36 เรื่องมีคลิปครบ พร้อมปล่อย
สัปดาห์ 2:  A4 (ปล่อยจริงครั้งแรก) ──────────────┐
                                                  ↓ เริ่มมีข้อมูลจริง
เดือน 1:    B3 (feedback จริง) + B1 (เรือธง) + B2 (Auto-QA)
เดือน 2:    B4 (scout ฉลาด) + B5 (observability)
เดือน 3+:   C1-C4 (สเกล + ธุรกิจ)
```

**กฎ:** อย่าเพิ่มฟีเจอร์ต้นน้ำ (scout/idea) จนกว่า funnel ท่อนปลายจะไหลครบ — ตอนนี้คอขวดอยู่ปลาย ไม่ใช่ต้น

## 5. ความเสี่ยง + ข้อควรระวัง

- **Mac mini เป็น single point of failure** — ถ้าดับ ทุกอย่างไป Gemini (แพง/throttle) → A3 สำคัญ
- **โควต้า/ค่า API** — Imagen หมดแล้ว (ใช้ ComfyUI แทน), Gemini มี cap → คุมด้วย single-flight + ComfyUI ฟรี
- **ลิขสิทธิ์** — ต้องมี originality guard (B4) ก่อนสเกลการเผยแพร่
- **คุณภาพร่วงตอนสเกล** — Auto-QA (B2) ต้องมาก่อนผลิตขนาน (C1)
- **Phase 4-5 ติด credential** — เป็น `[decision]` ของคุณ ไม่ใช่เรื่องโค้ด → ตัดสินใจเลือกแพลตฟอร์มเร็วเท่าไรยิ่งดี
